:diagnostics_1_plug:
Dieses Dienstprogramm wird zur Fehlerbehebung verwendet. Es wird alle Systemhardwareinformationen und Konfigurationsdateien sammeln und diese Dateien in eine einzelne ZIP Datei packen, welche lokal gespeichert werden kann.
Anschliessend kann die Datei in der Korrespondenz mit Limetech oder dem Unraid forum einbezogen werden.

Dies hilft anderen, schnell interne Informationen Ihres Systems zu bekommen und Ihnen besser bei Ihrem Problem zu helfen. Die folgenden Informations-
und Konfigurationsdateien werden gesammelt:
:end

:diagnostics_2_plug:
+ */config*
  <br>alle *\*.cfg Dateien*, *go* Datei und die *super.dat* Datei speichern. Diese sind Konfigurationsdateien.
+ */config/shares*
  <br>alle *\*.cfg* Dateien speichern. Diese sind Benutzerfreigaben Einstellungsdateien.
+ *Syslog Datei(en)*
  <br>Aktuelle *syslog* Datei und allfällige vorherig existierende *syslog* Dateien speichern.
+ *System*
  <br>Ausgabe der folgenden Befehle speichern:
  <br>lsscsi, lspci, lsusb, free, lsof, ps, ethtool & ifconfig.
  <br>Anzeige der iommu Gruppen.
  <br>Anzeige der Kommandozeile Parameter (z.B. pcie acs override, pci stubbing, etc).
  <br>System Variablen speichern.
+ *SMART Bericht*
  <br>Ein SMART Bericht von jedem individuellen Laufwerk, die sich im System befindet, speichern.
+ *Docker*
  <br>*docker.log*, *libvirtd.log* und *libvirt/qemu/\*.log* Dateien speichern.
:end

:diagnostics_3_plug:
Ein Klick auf **Download** wird den Sammelprozess starten und weist den Browser an, die zip Datei lokal zu speichern.

*Keine persönliche Informationen wie Benutzernamen, Passwörter oder andere nicht oben aufgelisteten Dateiinhalte werden von Unraid OS
einbezogen; Servername, IP Adresse und andere Benutzerfreigaben* **werden** *miteinbezogen.*

*Beachten Sie, dass Plugins von Drittanbietern **möglicherweise** persönliche Informationen in pluginspezifischen Konfigurationsdateien
und/oder System Log gespeichert werden können.*

*Benutzen Sie* **Diagnosik anonymisieren** *, wenn die Diagnosikdateien in einem öffentlichen Forum publiziert werden. In privater Kommunikation 
mit Limetech ist es empfohlen, diese Einstellung abzuwählen und alle Informationen unverändert zu erfassen.*
:end

:newconfig_plug:
Dies ist ein Dienstprogramm, um die Array Laufwerk Konfiguration zurückzusetzen, sodass alle Laufwerke
als "Neu"e Laufwerke erscheinen, als wäre es ein neuer Server.

Dies ist nützlich, wenn mehrere Laufwerke hinzugefügt oder entfernt wurden und Parity 
Datenwiederherstellung basierend auf der neuen Konfiguration erwünscht ist.

*'Aktuelle Aufgaben aufrechterhalten'* Auswahl benutzen, um gewünschte Laufwerkslots zu populieren, ***nachdem*** das Array zurückgesetzt wurde. 
Standardmässig werden **keine** Laufwerke populiert.  

**BENUTZEn SIE NICHT DIESES DIENSTPROGRAM UNTER DEM EINDRUCK, DASS ES EIN FEHLERHAFTES LAUFWERK WIEDERHERSTELLT** -
es wird die gegenteilige Wirkung haben, welches es **unmöglich** macht, ein existierendes fehlerhaftes Laufwerk wiederherzustellen - Sie wurden gewarnt!
:end

:newperms_1_plug:
Dies ist eine notwendige einmalige Aktion, welche nach der Aktualisierung von einer pre-5.0 Unraid Server Version zu Unraid v5 oder 
einer späteren Version stattfindet.
Wählen Sie **ALLE LAUFWERKE** aus, wenn diese einmalige Aktion ausgeführt wird.

Dieses Dienstprogramm ist auch nützlich, um die Standardberechtigungen/-eigentümer von Dateien
und Verzeichnissen wiederherzustellen, wenn von einer Active Directory zu einer 
non-Active Directory Integration zurückgewechselt wird.

Dieses Dienstprogramm startet einen Hintergrundprozess, welches über jedes Daten- und Cachelaufwerke und ausgewählten 
Benutzerfreigaben geht und Verzeichnis Eigentümer zu * nobody/users* (d.h. uid/gid zu 99/100) ändert und setzt Berechtigungen wie folgt:
:end

:newperms_2_plug:
Für Verzeichnisse:
  drwxrwxrwx

Für lesen/schreiben Dateien:
  -rw-rw-rw-

Für schreibgeschützte Dateien:
  -r--r--r--
:end

:newperms_3_plug:
Ein Klick auf **Start** öffnet ein neues Fenster und startet den Hintergrundprozess.

Das Schliessen des Fensters bevor Abschluss des Prozesses wird den Hintergrundprozess terminieren - also bitte nicht machen. Der Prozess kann eine lange Zeit
in Anspruch nehmen, wenn viele Dateien vorhanden sind.

Beachten Sie, dass dieses Tool möglicherweise Docker container negativ beeinflussen könnte, wenn die appdata Share vom Tool miteinbezogen wird.
:end

:registration_1_plug:
Der Server wird nicht nutzbar sein, bis eine Registrierungslizenz gekauft oder eine kostenlose 30-Tage *Probelizenz* 
installiert wurde. Eine *Probelizenz* beinhaltet alle Funktionalitäten einer *Pro* Registrierungslizenz.

Registrierungslizenzen sind an der Seriennummer des USB Flash Bootgeräts (GUID) gebunden. Bitte benutzen Sie ein hochwertiges Gerät
eines reputablen Unternehmens von mindestens 1GB.

Hinweis: USB Kartenlesegeräte sind generell **nicht** unterstützt, weil bei den meisten keine einmalige Seriennummer vorliegt.
:end

:registration_2_plug:
+ Bitte stellen Sie sicher, dass Ihre [Serverzeit][DateTime] auf 5 Minuten genau geht.
+ Bitte stellen Sie sicher, dass ein [DNS Server](DateTime) gesetzt ist.
:end

:registration_3_plug:
Ihre *Probelizenz* beinhaltet alle Funktionalitäten und Gerätunterstützungen einer *Pro* Registrierungslizenz.

Nachdem die *Probelizenz* abgelaufen ist, wird der Server **noch immer normal funktionieren**, bis das Array das nächste Mal **gestoppt** wird.

An diesem Punkt können Sie entweder eine Registrierungslizenz erwerben oder eine *Probelizenzverlängerung* beantragen.
:end

:registration_4_plug:
Um Unraid OS weiterhin benutzen zu können, können Sie eine Registrierungslizenz erwerben. Alternativ können Sie eine *Probelizenzverlängerung* beantragen.

Die meisten *Probelizenz* Verlängerungsanträge werden sofort bearbeitet, aber bitte erlauben Sie bis zu einem Werktag, um Ihren *Probe* Verlängerungslizenz zu erhalten.
:end

:registration_5_plug:
Es ist nicht möglich, eine *Probelizenz* mit einer existierenden Unraid OS Installation zu verwenden.

Es ist möglich, eine Registrierungslizenz dem USB Flash Gerät entsprechend zu erwerben, um die vorhandene Installation weiterhin verwenden zu können.

Für weitere Informationen, bitte den [Support Kontaktieren](https://lime-technology.com/contact).
:end

:registration_6_plug:
Es scheint als fehle Ihre Registrierungslizenzdatei oder sie ist korrupt. Die Lizenzdatei sollte unter dem
[config](/Registration/Browse?dir&#61;/boot/config) Verzeichnis auf Ihrem USB Flash Bootgerät zu finden sein.

Falls Sie keine Backup Kopie Ihrer Registrierungslizenzdatei haben, [Support Kontaktieren](https://lime-technology.com/contact).

Falls dies eine *Probeinstallation* war, können Sie eine Registrierungslizenz erwerben.
:end

:registration_7_plug:
*Probeinstallationen* sind nur mit dem ursprünglich registrierten USB Flash Gerät valide.

Um weiterhin diese Installation mit dem USB Flash Gerät zu verwenden, können Sie eine Registrierungslizenz erwerben.
:end

:registration_8_plug:
Die Registrierungslizenzdatei stimmt nicht mit dem USB Flash Bootgerät überein.
Bitte kopieren Sie die korrekte Lizenzdatei in das [config](/Registration/Browse?dir&#61;/boot/config) Verzeichnis
auf Ihrem USB Flash Bootgerät. Falls Sie keine Backup Kope Ihrer Registrierungslizenzdatei haben, [Support Kontaktieren](https://lime-technology.com/contact).

Falls Sie Ihre Registrierungslizenz mit einer neuen Lizenz zu diesem USB Flash Gerät verbinden möchten, klicken Sie unten auf Lizenz Ersetzen.
Eine Originallizenz kann jederzeit ersetzt werden. Danach kann eine Ersatzlizenz erst nach Ablauf eines Jahres erneut ersetzt werden.
Wenn Sie eine andere Ersatzlizenz früher benötigen, [Support Kontaktieren](https://lime-technology.com/contact).

**Hinweis:** Das Ersetzen einee Registrierungslizenz führt dazu, dass die vorherige USB Flash GUID dauerhaft auf die *schwarze Liste* gesetzt wird.
:end

:registration_9_plug:
Dieses USB Flash Bootgerät wurde auf die *schwarze Liste* gesetzt. Dies kann als Ergebnis der Übertragung Ihrer Registrierungslizenz 
an ein USB Flash Ersatzgerät auftreten und Sie werden derzeit von Ihrem alten USB Flash Gerät gebootet.

Ein USB Flash Gerät kann auch auf die *schwarze Liste* gesetzt werden, wenn es keine Seriennummer gibt oder wenn wir feststellen, dass die
Seriennummer nicht eindeutig ist (dies ist bei USB Kartenlesegeräten üblich).

Für weitere Informationen, bitte [Support Kontaktieren](https://lime-technology.com/contact).
:end
